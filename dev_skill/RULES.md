# RULES — dev_skill 운영 룰(초안)

이 문서는 dev_skill에서 적용할 운영 규칙의 초안입니다. 모든 개발자(사람)와 AI 에이전트는 아래 규칙을 준수해야 합니다.

1) 승인 원칙
- 외부 영향(텔레그램 전송, 대량 크롤링, 프로덕션 배포 등)은 명시적 승인 없이는 실행하지 않습니다.
- 승인 방법: Jira 이슈에 '진행해줘' 코멘트를 작성하여 승인한다.

2) 감사 로그
- 모든 AI 액션은 다음 필드를 포함한 로그를 남깁니다:
  - agent_id, action, input_hash, output_hash, related_issue, human_approver (nullable), timestamp
- 로그는 최소 90일 보관합니다.

3) 위험 차단
- robots.txt가 금지하거나 ai-legal이 '금지'로 플래그한 소스는 자동 수집 금지.
- PII가 감지되면 즉시 수집 중단 및 ai-legal 알림.

4) 테스트 통과
- PR 병합 전 필수: lint, unit tests, security scan 통과.

5) 점진적 권한 확대
- AI 권한은 단계적으로 부여합니다. 프로덕션 자동화는 별도 승인 절차 필요.

6) 모니터링/에스컬레이션
- LLM 실패, 수집 실패 등 반복 오류는 ai-ops에게 자동 에스컬레이션합니다.

7) 보존/보안
- 감사 로그와 원시 스냅샷은 접근 제어 하에 암호화 보관합니다.
