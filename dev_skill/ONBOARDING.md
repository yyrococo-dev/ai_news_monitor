# ONBOARDING — dev_skill

이 문서는 dev_skill에 포함된 AI 역할과 사람 역할의 책임을 정리한 문서입니다. 모든 역할은 프로젝트 규칙을 준수해야 하며, 외부 영향 작업은 인간의 승인이 필요합니다.

AI 역할(Agent) 정의

- ai-dev
  - 책임: 코드 변경 제안(PR 생성), 단위테스트 추가, 정적분석 수행
  - 권한: 브랜치 생성, PR 생성
  - 제한: main 병합은 사람 승인 필요

- ai-integrator
  - 책임: 통합 테스트 수행, 외부 연동(모의) 검증
  - 권한: 스테이징 통합 테스트 실행

- ai-qa
  - 책임: E2E 시나리오 자동 실행, 회귀 테스트 관리
  - 권한: 테스트 환경에서 시나리오 실행

- ai-ops
  - 책임: 스테이징 배포 자동화/모니터링, 프로덕션 배포 제안
  - 권한: 스테이징 배포 자동 실행, 프로덕션은 사람 승인 후 실행

- ai-legal
  - 책임: robots.txt/TOS 자동 검사 및 리스크 분류
  - 권한: 스크랩 허용/금지 권고(최종 결정은 사람)

- ai-notifier
  - 책임: 알림 초안 생성(텔레그램/메일), 요약 작성
  - 권한: 초안 작성(자동 전송은 사람이 허용 시 가능)

휴먼 인 더 루프(HITL)
- 외부 영향(텔레그램 전송, 프로덕션 배포, 대량 크롤링 등)은 반드시 사람의 명시적 승인(예: Jira의 "진행해줘" 코멘트) 이후에 실행됩니다.

감사 및 기록
- 모든 AI 행동은 감사 로그에 기록되어야 합니다. 로그 포맷은 RULES.md 참고.
