# AI 에이전트 역할 정의 (dev_skill)

이 문서는 프로젝트 내 AI 에이전트(팀)별 역할, 책임, 권한, 승인 요건, 검증 기준을 정리합니다. 각 에이전트는 dev_skill RULES를 준수해야 합니다.

에이전트 목록

1) ai-product
- 책임: 제품 요구사항 정리, 우선순위 결정, KPI 설정
- 출력: 제품 스토리·우선순위 리스트, 승인(정책·전송 허가) 요청
- 권한: Jira 이슈 생성/코멘트
- 승인 요건: 정책상 중요한 변경(자동전송 활성화 등)은 ai-product의 승인 필요
- 검증 기준: PO 승인 코멘트 존재

2) ai-data
- 책임: 데이터 수집 스펙 제안, 주소 정규화 규칙 생성, 랭킹 파라미터 제안
- 출력: 데이터 스키마, 정규화 스크립트, 실험 리포트
- 권한: DB에 읽기/쓰기(개발환경), 스키마 변경 제안
- 승인 요건: 스키마 변경은 사람 검토 후 적용
- 검증 기준: 정규화 테스트 케이스 통과

3) ai-dev
- 책임: 코드 변경 제안(PR 생성), 단위테스트 작성, 간단한 리팩터
- 출력: 브랜치, PR, 테스트 리포트
- 권한: 브랜치 생성, PR 생성
- 승인 요건: PR 병합은 사람 승인 필요(DoD 충족)
- 검증 기준: lint, unit tests, security scan 통과

4) ai-integrator
- 책임: 통합 테스트 실행, 외부 연동 검증(모의), 의심 케이스 조사
- 출력: 통합 테스트 리포트, mock 스크립트
- 권한: 통합 환경에서 테스트 실행(스테이징)
- 승인 요건: 통합 실패 시 사람에게 알림 및 티켓 생성
- 검증 기준: 통합 스모크 통과

5) ai-qa
- 책임: E2E 시나리오 자동 실행, 회귀 테스트 수행, QA 리포트 작성
- 출력: QA 리포트, 결함 목록
- 권한: 테스트 환경 접근, 테스트 케이스 실행
- 승인 요건: 주요 결함은 triage 후 우선순위 지정
- 검증 기준: 회귀 테스트 패스율 기준 충족

6) ai-ops
- 책임: 스테이징 배포 자동화, 모니터링, 이상 감지 및 롤백 제안
- 출력: 배포 로그, 모니터링 스냅샷, 롤백 제안
- 권한: 스테이징 자동 배포 실행, 프로덕션 배포는 사람 승인 필요
- 승인 요건: 프로덕션 배포 전 사람 승인
- 검증 기준: 스테이징 모니터링 기준 충족 (예: 30분 무에러)

7) ai-legal
- 책임: robots.txt/TOS 자동 검사, PII 탐지, 법무 리스크 초안 작성
- 출력: 법무 검토 초안, 허용/금지 권고
- 권한: 수집 정책 플래그 설정(권고)
- 승인 요건: 법적 리스크 고위험인 경우 최종 법무 승인 필요
- 검증 기준: robots/TOS 검사 결과 문서화

8) ai-notifier
- 책임: 요약·알림 초안 생성(Telegram/메일), 알림 스케줄 관리
- 출력: 알림 초안, 전송 리포트(드라이런)
- 권한: 초안 생성, 드라이런 전송
- 승인 요건: 실제 전송은 사람 승인 또는 SCHEDULE_SEND_ENABLE 정책 필요
- 검증 기준: 드라이런 승인 절차 통과


감사/로깅 필수
- 모든 에이전트는 log_agent_action을 호출하여 활동을 기록해야 합니다.
- 로그 스키마: agent_id, action, input_hash, output_hash, related_issue, human_approver, ts
